# omem-web v0.2.0 迭代设计文档

## 目标
聚焦数据安全、性能、UI用户体验三大方向，优化现有功能，补齐个人中心缺失。

---

## 1. 交互体验优化

### 1.1 替换原生 confirm
**现状**：2处使用原生 `confirm`/`window.confirm`
- `settings-page.tsx` 重置 Vault 密码
- `spaces.tsx` 删除空间

**方案**：使用 shadcn/ui `AlertDialog` 组件替代
- 标题 + 描述 + 取消/确认按钮
- 确认按钮使用 `destructive` variant

### 1.2 统一 Toast 系统
**现状**：
- `settings-page.tsx` 使用 `toast from "sonner"` ✅
- `spaces.tsx` 使用自定义 `useToast()` hook ❌
- `analytics.tsx` 使用自定义 `useToast()` hook ❌

**方案**：全站统一使用 `sonner` toast
- 删除/简化自定义 `toast-provider.tsx`
- 所有页面统一 `import { toast } from "sonner"`
- 统一 toast 样式：success/error 带描述

---

## 2. 登录页重构

### 2.1 字段简化
**现状**：name + apiKey + apiUrl（3个字段）
**方案**：
- 只保留 **API Key**（必填）
- **API URL** 折叠为可选高级设置（默认当前域名）
- 去掉 **账号名称** 字段

### 2.2 自动获取空间名
**现状**：登录时只调用 `/health`，name 是用户手填的
**方案**：
- 验证 `/health` 通过后，自动调用 `GET /v1/spaces`
- 找到 `space_type === "personal"` 的空间
- 将 `space.name` 作为用户显示名存储
- 如果获取失败，使用 API Key 前8位作为 fallback

### 2.3 视觉优化
- 更简洁的卡片式登录框
- API Key 输入框增加可见性切换（显示/隐藏）
- 已保存账号列表更紧凑

## 3. Header 重构

### 3.1 头像下拉菜单
**现状**：头像纯展示，旁边只有一个退出按钮
**方案**：点击 Avatar 弹出 DropdownMenu
- 显示当前空间名 + API Key（脱敏）
- 分隔线
- 「个人中心」→ /profile
- 「系统设置」→ /settings
- 「退出登录」→ logout

### 3.2 显示优化
- 小屏时只显示空间名（去掉"未登录"fallback）
- 空间名过长时截断

## 4. 个人中心页面（新增）

### 4.1 路由
- `/profile` → ProfilePage
- Sidebar 增加「个人中心」入口（User 图标）

### 4.2 页面结构
```
个人中心
├── 当前空间信息卡片
│   ├── 空间名称（大标题）
│   ├── 空间类型（Badge）
│   ├── 空间 ID
│   └── 成员数量
├── 记忆统计概览
│   ├── 总记忆数
│   ├── 今日新增
│   └── 空间数
├── 账号管理
│   ├── 当前 API Key（脱敏显示 + 复制按钮）
│   ├── API URL
│   └── 多账号快速切换列表（从 auth store）
└── 数据安全
    ├── 退出时清除所有数据
    └── 导出本地配置
```

### 4.3 数据获取
- `GET /v1/spaces` → 空间信息
- `GET /v1/stats` → 记忆统计
- auth store → 账号信息

## 5. 数据安全优化

### 5.1 API Key 存储
- 继续存 sessionStorage（页面关闭自动清理）
- 绝不存 localStorage 或 cookie
- 内存中也不长期保留（刷新页面需重新登录）

### 5.2 API Key 显示脱敏
- 规则：`前4位 + **** + 后4位`
- 示例：`c60b****75a`
- 提供复制按钮（复制完整 Key）

### 5.3 退出清理
- 清理 sessionStorage 所有 omem 相关数据
- 清理内存中的敏感状态
- 重定向到登录页

---

## 6. 性能优化

### 6.1 记忆列表虚拟滚动（预留）
- 当前数据量 376 条，暂时不需要
- 但代码结构预留虚拟滚动接入点

### 6.2 路由懒加载
- 使用 `React.lazy()` + `Suspense` 懒加载各页面
- 减少首屏 JS 体积（当前 index.js 949KB）

### 6.3 图片/资源优化
- 已有字体文件按需加载
- 无需额外优化
